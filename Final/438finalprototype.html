Final Project

Final Project Prototype

Overview and Usage

    In my game, I replicated a two dimensional basketball game which awards the user points at 10 point increments, and once the user reaches 40 points the player wins and can choose to play again. To play the game, you can click on the screen, and the ball will follow the mouse. In order to get points, you must aim the ball into the hoop or bounce off the wall.

Development Process

  1. I first changed the basketball to look like a mouse and changed the background and floor to be pink.
  2. Next I followed the Firebase tutorial from class to set up a basic firebase.js.
  3. For my next step I altered the function sendMessage in my firebase.js to output a score equivalent to the finalScore when the timer hits zero in my index.js.
  4. Next I had to change my index.js so that it included a start up screen and a closing screen. On top of this I added a timer that would only begin when the player clicked.

Issue Deep-Dive

  • I ran into many issues for this final prototype, the first of which were making the timer. I fixed this issue by assigning a variable screen to have the value of 0, and depending on whether the player had clicked or not, that number would go up and signify a new page (with an if statement). My next, biggest, issue was that the score that would show up when the game ended was linked to my index.js therefore it wouldn't go to Firebase. To fix this, I used the function sendMessage(score) in firebase.js to send a score rather than message to firebase.

Ideas and Future

I am also ideating for something in the game that might cause penalty for the user, but I have not thought about it yet. If I don't do that, then it is likely that I will link a randomized function to the net to make it change location.
